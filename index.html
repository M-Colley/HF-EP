Human‑Factors Extension Pack for SUMO

HF‑EP unifies five research prototypes for human factors in traffic simulation into one dependable add‑on for SUMO, with a Unity viewer for studies with people. Focus is stability, not new scientific models.

Contact, m.colley (at) ucl.ac.uk

12‑month project API stability Cross‑platform CI

Project summary

Traffic simulators often omit people and interfaces. Our group built focused prototypes in SUMO and Unity. HF‑EP merges them under one namespace with a stable Python API, packaging, tests, documentation, and governance so UK labs and courses can adopt with confidence.

Unified API

Agents, sensors, interface widgets, scenarios, logging

Releases & packaging

Versioned schemas, signed builds, Docker, Conda, PyPI

Docs & training

Tutorials and UK scenario packs, educator kits and clinics

Unity bridge

Timing control and headless visualiser for teaching labs

Approach

Seven work packages deliver a stable add‑on and a community path.

Architecture and unification

  • Core abstractions, environment, agent, sensor, interface, scenario, collector
  • Config schema with validation and stamps
  • Compat layer for legacy scripts

Reliability and release

  • Unit, property, integration, visual regression
  • Static checks, signed packages, Docker and Conda, PyPI

Docs and training

  • Versioned MkDocs site and API pages
  • Quickstart, intermediate, advanced tutorials
  • UK scenarios and educator kit

Maintainability features

  • Scenario DSL and schema
  • Structured collectors, Parquet and Arrow
  • Profiling and caching, benchmarks
  • Unity bridge protocol and headless visualiser

Community and sustainability

  • Mentored pull requests, clinics, sprints
  • Governance, code of conduct, RFCs, LTS policy
  • Case studies, educator packs, impact brief

Team and roles

  • PI, Mark Colley, architecture and API, governance, community, release decisions
  • Research Software Engineer, implementation, CI and packaging, tests, performance, releases
  • Advisory panel, quarterly review of roadmap and adoption
  • Contributors, PhD and MSc students, tutorials and examples

Objectives

  • Unify prototypes into one package with a stable API
  • Freeze versioned data schema with validators and ranges
  • Publish runnable notebooks and screencasts that reproduce results
  • Refactor ScootAR into a general Unity viewer with low skew
  • Grow community capacity and add external maintainers
