2. Project

Short description

The Human Factors Extension Pack (HF-EP) for the open source traffic simulation software SUMO, Simulation of Urban Mobility, enhances traffic simulation by consolidating five proven SUMO prototypes into a single, maintained add on. It delivers standardized Python APIs for traffic related agents, sensors, user interfaces, and data logging, versioned schemas, packaging on PyPI and Conda Forge, containers, and accessible documentation with tutorials and German scenarios. The focus is on software infrastructure, not a new end user app. Labs and developers gain a dependable foundation to model human factors in mobility systems.

92 / 100 words
Societal challenge

Mobility research often fails to account for human behavior at scale, which leads to unsafe or inequitable deployments and wasted public funds. Fragmented tooling means each lab reinvents ad hoc scripts, brittle integrations, and undocumented data formats, which block reproducibility and slow evidence based traffic policy. HF-EP targets this gap as a software infrastructure that many others can build on. It lowers the entry cost for universities, civic technology groups, and small consultancies to test, for example, pedestrian safety measures, automated vehicle interfaces, scooter policies, and vertiport plans before they reach streets or airspace. By standardizing APIs, schemas, and packaging, and by shipping clear documentation and teaching assets, the project converts scattered prototypes into technically reliable building blocks. The indirect audience is large because developers integrate libraries into widely used applications, while educators and public bodies benefit from repeatable studies and transparent methods.

142 / 175 words
Technical approach

HF-EP will be a typed Python package with a modular core and plugins. Core abstractions cover Environment, Agent, Sensor, Interface, Scenario, and Collector. A compatibility layer preserves legacy scripts. Configuration utilizes TOML and JSON Schema, incorporating semantic version stamps. CI on GitHub Actions builds and tests across Linux, Windows, and macOS, publishes signed wheels to PyPI and conda forge packages, and ships container images on the GitHub Container Registry, GHCR. Testing combines unit, property based, and integration tests with headless SUMO runs. Docs use MkDocs Material with API references from type hints and runnable tutorials. Releases follow semantic versioning with LTS branch and a deprecation policy. The design follows Prototype Fund principles, Data minimization, all data stays local, no cloud dependency. User friendliness, clear tutorials and examples enable participation without programming skills, German tutorials lower language barriers. Adaptability, modular architecture with plugins, TOML configs instead of code changes, scenario DSL for domain specific adaptations. Open Source, MIT license, open governance, public roadmap.

163 / 175 words
Current status and planned additions

Status, five released prototypes exist, PedSUMO, HUD-SUMO, UAM-SUMO, MM-SUMO, plus Generative Traffic Agents. They answer narrow research questions but suffer from version drift, duplicated utilities, sparse tests, and uneven documentation. Planned, merge into one namespace with a stable API, freeze a versioned data schema with validators, raise automated test coverage, ship cross platform packages and containers, add a headless mode, publish accessible docs with tutorials and scenarios for German traffic environments, adopt semantic releases with LTS, and install open governance and contribution pathways.

86 / 100 words
Similar approaches and differentiation

SUMO with TraCI offers base simulation, CARLA and CityFlow focus on vehicle autonomy or mesoscopic flows, MATSim targets activity based modelling. None provides a maintained human factors layer with stable APIs for user interface interventions, pedestrian intent exchange, micro mobility on sidewalks, and urban air mobility inside SUMO, together with university ready packs. HF-EP is infrastructure first beyond academic use. It standardizes schemas, packaging, testing, and governance so developers and educators can extend and trust one stack. It enhances reuse within the existing SUMO ecosystem rather than creating a parallel platform.

Secondary, Research Software Engineers in HCI and Transport. Tertiary, Policy labs and NGOs. Engagement, a public Slack, monthly clinics, mentored pull requests, workshops at CHI, AutoUI, and the SUMO User Conference, and educator kits integrated into MSc modules. Adoption is tracked with transparent metrics and migration guides.
